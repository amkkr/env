# 必ず遵守すべきルール

- 日本語で回答すること
- force-pushで過去のコミットログを消すようなコミット操作はしないこと
- コミットメッセージにCo-Authored-Byを含めないこと
- PRの説明にClaude Codeへの言及を含めないこと

## Git Rules

- **masterへの直接コミット禁止**: masterブランチに直接コミットしてはならない
- **ブランチ作成必須**: ファイルを操作する場合は必ずブランチを切ってから編集・コミットすること
- **ブランチ名のプレフィックス**: ブランチ名には必ず以下のプレフィックスをつけること
  - `feat/` - 新機能開発
  - `fix/` - バグ修正
  - `chore/` - ドキュメント更新や自動生成ファイルの更新など
  - `refactor/` - コードのリファクタリング
  - `update/` - ドキュメントや設定ファイルの更新
- **英語でのブランチ名**: ブランチ名は英語で記述すること
- **簡潔な命名**: ブランチ名は短く、わかりやすいものにする
- **直接マージ禁止**: 開発ブランチを直接masterにマージしてはならない
- **Pull Request必須**: マージは必ずGithubにpushして、Pull Requestを通して行うこと
- **rebase禁止**: git rebaseコマンドは使用しない。コンフリクト解決時はgit mergeを使用すること
- **PRテンプレート使用**: Pull Requestの内容は`.github/pull_request_template.md`を元に作成すること
- **PRタイトルのプレフィックス**: PRタイトルにはブランチ名と同様のプレフィックスをつけること（例: `update: CLAUDE.mdにルールを追加`、`feat: ユーザー認証機能を追加`）

### ブランチ名の例

```bash
# 良い例
feat/user-authentication
fix/login-error
chore/update-dependencies
refactor/improve-performance
update/claude-md

# 悪い例
new-feature
修正
my-branch
```
